# decisions.md — Ключевые архитектурные решения

> Это решения принятые намеренно. Не менять без явного обсуждения.

## БД и данные

**КСИ** — дерево с `parentId` в PostgreSQL. Загружать JSON-справочник через `prisma/seed.ts`.
Не дублировать справочник в других таблицах — только ссылки через `ksiNodeId`.

**Файлы** — только метаданные в БД (`s3Key`, `fileName`, `mimeType`, `size`).
Доступ через pre-signed URL от Timeweb S3 (TTL: 1 час). Прямых URL нет.

**Фото** — модель `Photo` с полиморфной связью (`entityType` + `entityId`).
Позволяет прикреплять фото к любой сущности без отдельных таблиц связей.

**Дефектовка** — аналогично `Photo`: `entityType` + `entityId` + аннотации в JSON.
`annotations: [{type: 'arrow'|'circle'|'text', x, y, ...}]`

**Multi-tenancy** — в КАЖДОМ Prisma-запросе фильтрация по `organizationId` из сессии.
Row Level Security в PostgreSQL как дополнительный уровень.

**Пагинация** — обязательна для: WorkRecord, Material, ExecutionDoc, Ks2Act, Photo, Defect.
Лимит 50 по умолчанию, максимум 200. Возвращать `{ data, total, page, limit }`.

## PDF и документы

**PDF-генерация** — Handlebars шаблоны в `/templates/*.hbs` → Puppeteer → Timeweb S3.
Шаблоны кэшировать как Promise на уровне модуля (не readFileSync в теле функции!).

**docx-шаблоны** — docxtemplater + pizzip, плейсхолдеры `{field}`.
Конвертация docx→PDF: LibreOffice headless. Превью: mammoth.js (docx→HTML).

**Иерархия генерации при наличии override**:
```
overrideHtml (если есть) → рендерить напрямую в Puppeteer
↓ нет
overrideFields (если есть) → подставить в шаблон
↓ нет
стандартный шаблон Handlebars
```

## Парсинг смет

XML из Гранд-Сметы → xml2js (детерминированно, без GPT).
Excel и PDF → YandexGPT с температурой 0.1 (чанки по 20 строк).
Strategy "Skip & Mark" — при ошибке чанка продолжать, вставлять предупреждение.
ВСЕГДА показывать экран предпросмотра перед записью в БД.
Gemini fallback при недоступности YandexGPT.

## Workflow согласования

`ApprovalRoute` + `ApprovalStep` — переиспользовать для всех типов документов:
ИД (АОСР, ОЖР), СЭД-документы, Задания на ПИР, Документы ПИР, КС-2/КС-3.
При подписании текущего шага → автопродвижение к следующему.
Уведомление следующему участнику через BullMQ + Email.

## Чат

Socket.io сервер — ВСЕГДА отдельный процесс на порту 3001.
Не интегрировать в Next.js API Routes (конфликт с serverless-моделью).
Сообщения хранить в PostgreSQL (не только в памяти).
При недоступности сокета → показывать кнопку «Обновить», не бросать ошибку.

## ЭЦП

Абстракция `SignatureProvider` — интерфейс для разных провайдеров.
Начать с КриптоПро CSP через REST API шлюз.
Не привязываться к конкретному провайдеру в бизнес-логике.

## ИСУП Минстроя

Отдельный сервис-адаптер `lib/isup/`.
Формирование XML-пакетов по спецификации ИСУП.
Отправка через BullMQ-задачу с retry-логикой (3 попытки).
Маппинг КСИ → коды ИСУП: таблица `IsupCodeMapping`.

## Диаграмма Ганта

`gantt-task-react` (MIT) для MVP.
Данные: `GanttTask` (planStart/End, factStart/End, progress, isCritical).
Версионирование: `GanttVersion`. Связь с `WorkItem` через `workItemId`.
При масштабировании → мигрировать на `dhtmlx-gantt`.

## PWA

Кэшировать: статику, последние 10 открытых документов, справочник КСИ.
НЕ кэшировать: API-запросы с мутациями (POST/PUT/DELETE).
Настройка через Serwist (`@serwist/next`) — форк Workbox с поддержкой Next.js 14+.

## ФЗ-152

Все данные пользователей — только на Timeweb Cloud (РФ).
Не использовать зарубежные CDN для персональных данных.
YandexGPT — через Yandex Cloud API (серверы РФ ✅).
